Vehicles In Lot Methods

The vehiclesInLot collection holds documents that model all vehicles currently in a given parking lot.


/api/vehiclesInLot

Vehcles in the lot are stored as BSON documents. The 'plate' field is required when creating a vehicle document.
GET: Return array with all vehicles in lot collection (VEHICLES_IN_LOT_COLLECTION)
POST: add new vehicle to lot from JSON passed in request body

You can use CURL to access the API. Here is an example of adding a new vehicle to the lot:

$ curl -X POST -H "Content-Type: application/json" -d '{"plate":"000 000", "registered":""}' https://ancient-inlet-70380.herokuapp.com/api/vehiclesInLot

If you use that code, it will create a document with the bare minimum of data. If you pass in a value for 'registered' it will be set to ''


/api/vehiclesInLot/ID/:id

Upon creation, a document is given a unique id by the database. The '_id' field should never be set manually.
GET: Return document by id (Unique '_id' field generated by database)
PUT: Find and update document by id

You can use CURL to access the API. PUT requests were particularly hard to get working so here is an example:

$curl -X PUT -H "Content-Type: application/json" -d '{"registered":"true"}' https://ancient-inlet-70380.herokuapp.com/api/vehiclesInLot/ID/5ad69d2c734d1d7f2372bdd6

DELETE: Find and delete document by id

/api/vehiclesInLot/Plate/:plate

Because the 'plate' field is required, and pseudo-unique, we expose methods to perform CRUD operations based on the 'plate' field
GET: find and return document by plate. Returned as JSON.


Registered Vehicles Methods

The registeredVehicles collection holds the data for every vehicle registered (see: allowed) to park in a given lot.


/api/registeredVehicles

Registered vehicles are typically stored with a name, student id, make, model, color, and plate. However, to reduce inconvenience, only the 'plate' and 'studentID' fields are required.
GET: Return all documents in an array. Returned as JSON
POST: Register a new vehicle from JSON passed in request body

/api/registeredVehicles/ID/:id

Upon creation, a document is given a unique id by the database. The '_id' field should never be set manually.
GET: Return vehicle by id. Returned as JSON.
PUT: Update vehicle by id
$curl -X PUT -H "Content-Type: application/json" -d '{"plate":"000 000"}' https://ancient-inlet-70380.herokuapp.com/api/registeredVehicles/ID/5ad69d2c734d1d7f2372bdd6

DELETE: Delete vehicle by id

/api/registeredVehicles/Plate/:plate

Because the 'plate' field is required, and pseudo-unique, we expose methods to perform CRUD operations based on the 'plate' field
GET: Return vehicle by plate. Returned as JSON.
